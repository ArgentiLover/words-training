# Тренажёр слов

[Сайт](https://words-training.vercel.app/)

Приложение "Тренажёр слов" представляет собой простой веб-тренажёр для изучения английских слов с переводом на русский. Оно реализовано на Node.js с использованием Express и включает фронтенд на HTML, CSS и JavaScript. Все требования задания выполнены.

---

## Функционал

- **Express-сервер** с маршрутами для работы с данными и тренажёром слов.
- **Маршруты CRUD** для слов:

  ```
  GET /api/words         — получение всех слов, с поддержкой поиска через query-параметр ?search=...
  GET /api/words/:id     — получение слова по ID
  POST /api/words        — добавление нового слова (word и translation)
  PUT /api/words/:id     — обновление слова
  DELETE /api/words/:id  — удаление слова
  ```

- **Маршруты тренажёра**:

  ```
  POST /api/start  — начало тренировки, случайный порядок слов, обнуление очков
  GET  /api/train/next   — получение следующего слова для перевода
  POST /api/train/check  — проверка ответа пользователя, обновление очков
  ```

- **Обработка запросов**:
  - Используется `express.json()` и `express.urlencoded()` для парсинга тела запроса.
  - Параметры URL обрабатываются через `req.params`, query-параметры через `req.query`.
- **Собственный middleware**:
  - Логгер, который выводит метод запроса и URL каждого обращения.
- **Раздача статических файлов** через `express.static()` из папки `public`.
- **Логика разнесена по модулям**:
  - `routes/words.routes.js` и `routes/train.routes.js` для маршрутов.
  - `controllers/words.controller.js` для обработки данных и тренажёра.
- **Фронтенд функционал**:
  - Отображение случайного слова для перевода.
  - Ввод перевода пользователем.
  - Проверка правильности ответа, отображение верного перевода.
  - Отображение прогресса и очков.
  - "Победное окно" при завершении всех слов.
  - Возможность добавлять новые слова через форму.
  - Прогресс тренировки сохраняется в `localStorage`, включая очки и очередь слов.
- **Случайный порядок слов** в тренировке, чтобы каждый запуск был уникален.
- **Взаимодействие с API** осуществляется через fetch-запросы с фронтенда.
